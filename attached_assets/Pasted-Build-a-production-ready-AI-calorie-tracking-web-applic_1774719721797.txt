Build a production-ready AI calorie tracking web application similar to Cal AI using strictly Angular (latest version) for the frontend, Spring Boot (Java) for the backend, and PostgreSQL for the database. The application should allow users to sign up and log in securely using JWT-based authentication (with optional Google OAuth), and once authenticated, users should be able to upload images of their meals. The system must upload images directly to an object storage service (such as AWS S3 or equivalent) and only store the image URL in the database (do not store Base64 images). The backend should send the image URL to an AI vision model to extract nutritional data such as calories, protein, carbs, fats, health score, and confidence score, and must strictly validate that the AI response matches a defined JSON schema, retrying if necessary. All AI processing must happen asynchronously using background jobs so that the user interface is never blocked; while processing, the frontend should show a non-blocking indicator and allow navigation across the app. The dashboard should display daily calorie intake, remaining calories, and include analytics with daily, weekly, and monthly charts. Each meal entry should store image URL, nutrition data, timestamp, and user association. The backend must be stateless and designed for scalability, with clean REST APIs, proper DTO usage, pagination, indexed queries, and separation of concerns (authentication, meal handling, AI processing). The frontend should be mobile-responsive, fast, and use lazy loading where appropriate. Implement proper error handling, input validation, and secure all endpoints. Additionally, design the system to scale to a large number of users by avoiding blocking operations, optimizing database queries, and structuring the code in a way that supports future enhancements such as editing AI results, multi-food detection, and integration with wearable devices. First plan the architecture clearly, then implement step by step instead of generating everything at once.
